How does this work?

Prompt caching: Claude Code caches your conversation context so it doesn’t need to re-process it every turn. This cache has a time-to-live (TTL) — 5 minutes by default, or 1 hour if you enable extended caching.

Cache miss: When the cache expires (you were idle too long), Claude has to rebuild the entire cache from scratch. This is expensive — writing tokens to cache costs 12.5× more than reading from it.

Why big sessions cost more: Every turn reads your full conversation context. A session at 200K tokens reads 200K tokens per turn. At $0.50/MTok, that’s $0.10 just in cache reads — before any output.

The 5-minute rule: If you’re in a big session and need a break, either switch to 1-hour TTL so the cache survives longer breaks, or run /clear before stepping away so you don’t pay to reload a huge context.
